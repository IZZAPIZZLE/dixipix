# CLIENT-INFO — Context

## What this workspace is for
Client and project records. Text only. No images, no code.

## What lives here
- One file per client project (e.g. `yowie-bay.md`, `taren-point.md`)
- Each file contains: client name, project brief, deliverables, status, notes

## File naming
`[suburb-or-project-name].md` — lowercase, hyphenated

## Process
1. New project confirmed → create a new `.md` file here
2. Fill in client details, brief, what's needed
3. Update status as project moves forward

## What does NOT live here
No images. No code. No videos. Those go in CONTENT/ and CODE/.
